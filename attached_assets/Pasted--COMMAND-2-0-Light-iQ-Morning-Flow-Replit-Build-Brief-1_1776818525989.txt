# COMMAND 2.0 Light — iQ Morning Flow
## Replit Build Brief

---

## 1. Context (READ FIRST)

This is **not** a new project. You are working inside the existing `artifacts/flipiq-dashboard` workspace in this repo (`NewUI`). Do not scaffold a new app. Do not create a new folder outside `artifacts/flipiq-dashboard`. Do not install new dependency families — everything you need is already in `package.json` (React, Vite, wouter, shadcn/ui, Tailwind, lucide-react, framer-motion).

The existing app currently routes:
- `/` → `MyStats`
- `/adaptation-reports` → `AdaptationReports`

You will add a new top-level flow mounted at `/iq` that drives the entire COMMAND 2.0 Light experience.

**What you are building:** a seven-step guided daily workflow that replaces free navigation with a linear "iQ tells the AA what to do next" experience. The AA logs in, iQ walks them screen by screen, and the sidebar unlocks items as they progress. When they leave and come back, iQ resumes them at the exact step they left off.

---

## 2. Hard Design Rules (DO NOT VIOLATE)

1. **White / light theme only.** Background is `#f5f6f8` (matches `MyStats`). Sidebar is `bg-white` with a right border. No dark dashboards. No dark sidebars.
2. **Brand accent is orange `#F97316` (Tailwind `orange-500`).** All primary buttons, active nav states, active step indicators, and "iQ" branding use this orange. No green, no blue primaries.
3. **Font is the existing app font** (Plus Jakarta Sans via Tailwind `font-sans`). No new font imports.
4. **Do NOT embed the reference PDF screenshots as images.** Build every UI element as native React + Tailwind. Pixel-sharp borders, crisp text, no blurry assets.
5. **All buttons, inputs, and cards use consistent sizing** — pull from shadcn/ui primitives (`Button`, `Card`, `Input`, `Badge`) that already exist in `src/components/ui/`. Do not roll new button styles.
6. **Typography hierarchy**: orange bold for screen titles and section headers, dark gray (`text-gray-800`) for body, light gray (`text-gray-400`) for metadata, `text-gray-600` for labels.
7. **Never remove existing functionality.** `MyStats` and `AdaptationReports` must continue to work exactly as they do today.

---

## 3. The Flow (7 Screens)

The AA moves through these in order. Each screen has the same top bar layout (see §4) and unlocks the next sidebar item when complete.

| # | Route | Screen | Trigger to next |
|---|-------|--------|-----------------|
| 1 | `/iq` | Morning Check-in | Answers both questions |
| 2 | `/iq/tasks` | Today's Tasks summary | Clicks "Click here to get started" |
| 3 | `/iq/deal-review` | Deal Review (9 properties, 3 segments) | Clicks "Next Task" in top bar |
| 4 | `/iq/daily-outreach` | Daily Outreach — Campaigns | Clicks "Send Campaign" then "Next Task" |
| 5 | `/iq/priority-agents` | Daily Outreach — Call Priority Agents | Clicks "Next Agent" through all agents |
| 6 | `/iq/new-relationships` | Start New Relationship Building (high-propensity deals + iQ Property Intelligence) | Clicks "Next Deal" through all deals |
| 7 | `/iq/welcome-back` | Welcome Back (shown on return if flow was not completed same day) | Clicks "Click here to get started" → resumes at last incomplete step |

### 4.1 Sidebar (reuse existing `src/components/Sidebar.tsx`)

Modify the existing Sidebar to add a new mode: when the URL starts with `/iq`, the sidebar collapses to ONLY the `TODAY'S PLAN` section. Hide Revenue Pipeline, Find Deals, Insights, Tools sections entirely. The AA does not get to pick where to go.

Sidebar items in iQ mode:
- **iQ** (always shown, active on `/iq` and `/iq/tasks` and `/iq/welcome-back`)
- **Deal Review 3/9** (shown once user reaches screen 3; active on `/iq/deal-review`; shows `✓` when complete)
- **Daily Outreach 0/12** (shown once user reaches screen 4 OR 5 — they are sub-steps of the same sidebar item; use the active state when URL matches either)
- Items NOT yet reached are hidden entirely (not greyed out — invisible).

**Add a new sidebar footer item: `← Back to COMMAND`** above the user profile block. On click, navigate to `/` (MyStats). This is the only escape hatch.

### 4.2 Top Bar (new component: `src/components/iq/IqTopBar.tsx`)

Three props: `breadcrumb: string`, `nextTask: string | null`, `onNext?: () => void`.

Layout:
- **Left**: Breadcrumb in medium gray text (e.g., `Deal Review > 9 High Priority Deals > 3 Active & Off Market`)

No page-level H1 — the top bar IS the location indicator.

### 4.3 Task + Tip block (new component: `src/components/iq/TaskTipBlock.tsx`)

Two props: `task: string`, `tip: string`.

Renders directly under the top bar on screens 3–6:

- `Task:` label is orange bold, body is dark gray.
